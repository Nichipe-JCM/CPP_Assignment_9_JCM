# GameChat Project

언리얼 엔진 C++로 구현한 멀티플레이 채팅 기반 미니게임 프로젝트입니다.  
채팅 시스템 위에 게임 로직을 연동하여, 여러 플레이어가 같은 방에서 대화하고 특정 유저가 게임을 선택한 뒤 함께 플레이할 수 있도록 구현했습니다.

## 프로젝트 개요

이 프로젝트는 Unreal 기반 멀티플레이 환경에서 채팅 기능과 미니게임 진행 로직을 함께 구현하는 것을 목표로 했습니다.  
채팅 명령어를 통해 게임을 선택하고, 참가자를 모집한 뒤, 턴 기반으로 게임이 진행되도록 구성했습니다.

## 주요 기능

- 멀티플레이 채팅 기능
- 닉네임 입력 및 입장 처리
- 채팅 명령어 기반 게임 선택
- 참가자 모집 및 상태 관리
- 턴 기반 진행
- 턴 타이머 UI
- Wordle / 숫자야구 게임 플레이
- 게임 종료 후 승리 / 무승부 판정 및 리셋

## 사용 명령어

- `!game` : 게임 선택모드 진입
- `!wordle` : Wordle 게임 선택
- `!baseball` : 숫자 야구 게임 선택
- `!join` : 현재 모집 중인 게임 참가

## 구현 과정

1. 멀티플레이 기반 클래스 구조를 먼저 설계하고 헤더를 작성.
2. 멀티플레이 채팅 로직과 UI를 우선적으로 구현 및 테스트.
3. 이후 미니게임 로직을 구현.
4. 게임 로직을 멀티플레이 환경에 연동.
5. 마지막으로 최소한의 UI 폴리싱과 버그 수정 진행.

## 클래스 구성

- `GameMode` : 전체 게임 진행 로직 관리
- `GameState` : 방 상태 및 게임 상태 동기화
- `PlayerController` : 입력 처리 및 UI 연동
- `PlayerState` : 플레이어별 상태 관리
- `UI Widget` : 채팅창, 닉네임 입력창, 게임 오버레이 표시

## 구현하면서 신경 쓴 점

- 서버 권한 기반으로 게임 진행 로직을 관리했습니다.
- 멀티플레이 환경에서 상태 동기화가 자연스럽게 이루어지도록 구성했습니다.
- 채팅 기능과 게임 로직이 서로 충돌하지 않도록 구조를 분리했습니다.
- 턴 진행과 타이머 UI를 통해 플레이 흐름이 보이도록 했습니다.

## 느낀 점

이번 과제를 통해 언리얼 멀티플레이 구조에서  
서버, 클라이언트, 상태 동기화, UI 연동이 어떻게 연결되는지 직접 구현하며 이해할 수 있었습니다.  
또한 단순 채팅 기능을 넘어서 게임 로직을 멀티플레이에 연결하는 과정을 경험할 수 있었습니다.
